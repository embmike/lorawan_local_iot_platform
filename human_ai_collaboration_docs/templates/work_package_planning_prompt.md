# Work Package Planning Prompt

Use this prompt with ChatGPT to prepare one bounded work package before ChatGPT Codex executes anything.

## Prompt

Create one bounded work package for the next engineering step.

The package should make the following visible:

- package goal
- non-goals
- affected files or areas
- required inputs from the frame
- required BDD features, scenarios, or equivalent state-based tests
- required unit tests and system tests
- expected documentation updates
- done criteria
- review path
- stop conditions that must return the work to clarification

Also state explicitly:

- what is already ready for execution
- what still needs clarification
- what still needs to be moved into the correct document or format before release
